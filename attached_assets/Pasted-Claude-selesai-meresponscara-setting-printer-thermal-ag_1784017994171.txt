Berikut panduan umum yang biasa dipakai:
1. Install Driver Printer Thermal

Sambungkan printer thermal ke komputer (USB/Bluetooth/LAN).
Install driver resmi dari pabrikan (misal: Epson TM-T20/T82, Xprinter, Zebra GK420, dll — cari "driver [merk+tipe printer] windows/mac").
Setelah terinstall, printer akan muncul di Devices and Printers (Windows) atau System Settings > Printers (Mac).

2. Atur Ukuran Kertas Sesuai Label/Struk

Buka Printer Properties > Preferences.
Set ukuran kertas custom sesuai label barcode Anda (misal 58mm x 40mm, atau ukuran struk 58mm/80mm).
Set orientasi dan margin ke 0 agar tidak terpotong.

3. Cetak dari Browser

Buka halaman barcode di website Anda.
Tekan Ctrl+P (Windows) atau Cmd+P (Mac).
Pilih printer thermal Anda sebagai tujuan cetak.
Di pengaturan lanjutan, matikan margin/header-footer, dan pastikan skala "100%" atau "Fit to page" sesuai kebutuhan.

4. Jika Hasil Cetak Kurang Presisi (Umum Terjadi)
Browser sering kurang akurat untuk mencetak barcode karena mengikuti tata letak halaman web, bukan ukuran label asli. Solusi yang lebih baik:

Gunakan software bridge seperti QZ Tray (gratis, open-source) yang bisa dipanggil dari JavaScript website untuk print langsung ke printer thermal via ESC/POS command — hasil lebih presisi dan bisa auto-print tanpa dialog print.
Kalau website berbasis sistem kasir/POS (WooCommerce, custom web app, dll), banyak yang pakai QZ Tray atau plugin serupa untuk printer thermal.

1. Ukuran Kertas di Driver Tidak Sesuai

Buka Control Panel > Printer & Scanner > [Printer Thermal Anda] > Printing Preferences.
Cek apakah Paper Size sudah diset custom sesuai label barcode Anda (misal 40mm x 30mm, 58mm x 40mm), bukan ukuran default seperti A4/Letter.

2. Scaling di Dialog Print Browser

Saat Ctrl+P, cek bagian More settings:

Scale: pastikan diset "Default"/"100%", JANGAN pakai "Fit to page" — ini sering jadi penyebab barcode mengecil/membesar otomatis mengikuti ukuran kertas driver.
Margins: set ke "None".
Paper size: pilih ukuran custom yang sudah dibuat di driver, bukan A4.



3. CSS Website Tidak Diset untuk Ukuran Label
Kalau Anda yang membuat halaman barcode di website, ukurannya harus didefinisikan eksplisit pakai @media print dan satuan fisik (mm/cm), bukan px atau %. Contoh:
css@media print {
  @page {
    size: 58mm 40mm;
    margin: 0;
  }
  body {
    width: 58mm;
    height: 40mm;
  }
}
Tanpa ini, browser akan menyesuaikan ukuran halaman ke default (biasanya A4), sehingga barcode tercetak kecil di pojok atau malah terpotong.
4. DPI Printer vs DPI Gambar Barcode
Kalau barcode-nya berupa gambar (bukan font/vector), pastikan resolusinya cukup tinggi (min. 203 DPI, sesuai printer thermal kebanyakan). Barcode dari gambar low-res akan tercetak buram atau ukurannya melar/menyusut tidak proporsional.

Penyebab paling umum untuk kasus ini:
Penyebab #1: Paper Size Driver ≠ Paper Size CSS (paling sering terjadi)
Browser me-render preview sesuai @page { size: 58mm 40mm } di CSS Anda, TAPI kalau paper size yang aktif di driver printer ternyata beda (misal driver masih set ke A4/Letter atau ukuran default lain), maka:

Browser terpaksa "menyusutkan" konten agar muat ke ukuran kertas driver → hasil jadi kecil.

Solusi: Pastikan ukuran kertas di driver printer (Printing Preferences) PERSIS SAMA dengan yang didefinisikan di CSS @page. Kalau CSS-nya 58mm x 40mm, driver juga harus diset custom paper size 58mm x 40mm — bukan dipilih dari dropdown default.
Penyebab #2: Scale Otomatis "Fit to Printable Area"
Beberapa driver printer thermal (terutama Windows generic/text driver atau driver merk tertentu) punya opsi tersembunyi seperti "Shrink to fit" atau "Scale to fit" yang aktif default. Ini akan menyusutkan hasil cetak walau preview browser sudah benar.
Solusi: Cek di Printer Properties > Advanced atau Preferences, cari opsi scaling dan set ke 100% / None / Actual Size.
Penyebab #3: Print Dialog Browser Sendiri Override
Saat Ctrl+P, cek ulang bagian Scale di dialog print — kadang browser (terutama Chrome) otomatis pilih "Fit to printable area" meskipun paper size sudah dipilih benar.
Solusi: Ubah manual ke "Custom" → 100%.

Cara cepat mengecek mana penyebabnya:

Buka dialog print (Ctrl+P).
Screenshot atau cek apakah Paper size yang terpilih sudah sesuai (bukan A4).
Cek nilai Scale — kalau bukan "100%" atau "Default", itu biang keroknya.

Kalau sudah dicek dan paper size + scale sudah benar tapi tetap kecil, kemungkinan besar drivernya sendiri yang override — solusi paling stabil adalah pakai QZ Tray untuk print langsung via ESC/POS command, karena ini bypass sepenuhnya print dialog browser dan drivernya jadi presisi 1:1 ke ukuran fisik.
